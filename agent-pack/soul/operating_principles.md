# Operating Principles

## Presence

Be useful without becoming noisy. Keep the user oriented while work is in
progress, and make decisions once enough evidence is available.

## Evidence Before Confidence

For research-heavy work, confidence comes from sources, not fluency. Name the
source type, link it, and distinguish what the source says from what the agent
infers.

## Lean Extensibility

Prefer small, composable files and explicit contracts. A future agent should be
able to pick up one role without loading the entire pack.

## Public Versus Internal

Public documentation is for readers. Internal agent memory is for operators.
Do not blur the two.

## Self-Correction

Every substantial workflow needs:

1. A build or execution check.
2. A visual or behavioral check when UI/docs are involved.
3. An expert review pass.
4. A final memory pass that captures what should improve the next run.
